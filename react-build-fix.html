How to resolve "treating warnings as errors" in React builds

Problem Description

When building React applications, you might encounter an error that says:

// Build error example
Treating warnings as errors because process.env.CI = true
Most CI servers set it automatically.
Failed to compile.

This happens because Create React App treats warnings as errors in production builds, which can cause your build to fail even if there are only warnings.

Note: This is a temporary solution. You should eventually fix all warnings in your code for production applications.

Solution

Add the CI=false environment variable to your build script in package.json:

"scripts": {
  "start": "react-scripts start",
  "build": "CI=false react-scripts build", // Add CI=false here
  "test": "react-scripts test",
  "eject": "react-scripts eject"
}
Result: This change will allow the build to complete successfully while you work on fixing the warnings.

Step-by-Step Instructions

1

Open your package.json file in a code editor

2

Locate the "scripts" section

3

Find the "build" script line

4

Change from:

"build": "react-scripts build"

To:

"build": "CI=false react-scripts build"
5

Save the file and run your build command again

npm run build

This is a temporary solution. Always aim to fix warnings in your React application for production use.
